This template is optimized for deployment on **Vercel** with **Vercel Postgres**.

## Deploy to Vercel

1. Push your repo to GitHub
2. Import the project on [vercel.com](https://vercel.com)
3. Add a Postgres database from the Vercel dashboard (Storage → Create → Postgres)
4. Vercel automatically sets `DATABASE_URL` — no manual config needed
5. Deploy

That's it. The build script automatically runs `prisma generate` and `prisma db push`.

## Environment Variables

### Required

| Variable | Description |
|----------|-------------|
| `DATABASE_URL` | PostgreSQL connection string (auto-set by Vercel Postgres) |

### Optional (can also be set via setup wizard)

| Variable | Description |
|----------|-------------|
| `NEXT_PUBLIC_WHOP_APP_ID` | Your Whop app ID |
| `WHOP_API_KEY` | Your Whop API key |
| `WHOP_WEBHOOK_SECRET` | Webhook signing secret |
| `SESSION_SECRET` | JWT signing secret (auto-generated if not set) |

## First Visit

After deploying, visit your site. If Whop isn't configured yet, you'll be redirected to the setup wizard at `/setup` which guides you through:

1. Entering your Whop app ID and API key
2. Connecting your plan IDs
3. Setting up webhooks

The first user to sign in via OAuth becomes the admin.

## Custom Domain

Add your custom domain in the Vercel dashboard under Settings → Domains. Update your Whop app's redirect URL to match:

```
https://your-domain.com/api/auth/callback
```

## Database Migrations

For schema changes in production:

```bash
# Generate migration
pnpm db:migrate

# Or push directly (for simple changes)
pnpm db:push
```
